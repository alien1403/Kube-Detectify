Kube-Detectify

TOOL TO IDENTIFY KUBERNETES ATTACKS

Container Administration
Command Abuse

Adversaries may abuse a container administration service to execute commands within a container. A container administration service such as the Kubernetes API server, or the kubelet may allow remote management of containers within an environment.

Mitigations you can make in order to keep your cluster safe:


- use read-only containers, read-only file systems
- make sure containers are not running as root by default
- enforce authentication and RBAC on the container service

Container
Deployment

Adversaries may deploy a container into an environment to facilitate execution or evade defenses. In some cases, adversaries may deploy a new container to execute processes associated with a particular image or deployment, such as processes that execute or download malware.

Mitigations you can make in order to keep your cluster safe:


- limit communication with the container service to local Unix sockets/remote access via SSH
- require secure port access to communicate with the APIs over TLS
- deny direct remote access to internal systems

Internal Image
Implanting

Mitigations you can make in order to keep your cluster safe:


- periodically check the integrity of images and containers used in cloud deployments
- use content trust models that require container images be signed by trusted sources
- limit permissions associated with creating and modifying platform images or containers
